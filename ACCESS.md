# 접근 권한 정리

## 1. 고객

- **관리자 페이지(/admin)**: ❌ 볼 수 없음 (미들웨어가 쿠키 없으면 /admin/login으로 리다이렉트)
- **디자이너 페이지(/groomer)**: ✅ 접근 가능 (디자이너 신청/로그인 화면만 노출, 대시보드는 비밀번호 필요)

## 2. 디자이너

- **디자이너 신청**: `/groomer/register`에서 신청
- **승인 후 비밀번호**: 관리자가 디자이너 관리에서 "비밀번호 부여" 후 사용 가능. 설정 직후 **전달 안내 창**에서 비밀번호·안내 문구 복사, 등록 전화번호가 있으면 **문자앱으로 열기**로 디자이너에게 전달할 수 있음
- **비밀번호 없음**: "비밀번호가 아직 부여되지 않았습니다. 관리자에게 문의해 주세요." 메시지
- **로그인 후**: 본인 예약·정산·프로필만 조회/수정 가능
- **관리자 페이지**: ❌ 접근 불가 (관리자 쿠키 없음)

## 3. 관리자

- **별도 비밀번호**: 최초 1회 설정, 이후 로그인
- **접근 범위**: 디자이너·예약·고객·정산·설정 등 전체 조회/수정
- **미들웨어**: `/admin` 접근 시 쿠키 확인, 없으면 `/admin/login`으로 리다이렉트

## 4. 서버 측 보호

| 경로 | 보호 방식 |
|------|-----------|
| `/admin`, `/admin/*` (login 제외) | 미들웨어: 쿠키 없으면 /admin/login 리다이렉트 |
| `/groomer`, `/groomer/*` | 클라이언트: 비밀번호 확인 후 대시보드 노출 |
| `/admin/login` | 공개 (로그인용) |

**참고**: `/groomer`는 미들웨어로 막지 않음. 디자이너 신청·로그인 화면은 공개되어야 하므로, 대시보드만 클라이언트에서 비밀번호로 보호함.
